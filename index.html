    RL-GMM 无监督聚类改进项目¶

    🎯 项目背景与意义¶

    本项目基于 OpenHUTB/nn/src/chap11_gaussian_mixture/GMM.py 进行小幅改进。
    改进目标是：在不改变 GMM+EM 核心流程的前提下，提升脚本的可配置性、可复现性和结果可保存性。

    ❓ 原始版本的使用痛点¶

    1. 关键参数主要写在脚本中，复现实验时不够方便。
    2. 图像以窗口展示为主，结果留存不够规范。
    3. 对数似然收敛过程缺少自动化日志文件。

    🛠️ 本次改进策略¶

    1. 增加命令行参数入口（样本数、成分数、迭代数、随机种子等）。
    2. 保留原算法主体，仅扩展运行和输出环节。
    3. 自动保存聚类图、收敛曲线和迭代日志。

    🚗 项目概述¶

    🧠 任务定义¶

    本实验属于无监督聚类任务：在没有人工标签的情况下，使用高斯混合模型（GMM）拟合样本分布，并通过 EM 算法迭代估计参数，输出样本聚类结果。
    实验数据由脚本自动生成的二维混合高斯样本构成。

    🔬 核心技术详解¶

    1. 📌 GMM 模型¶

    GMM 将数据看作多个高斯分布按权重混合生成。
    每个簇由权重 \( \pi_k \)、均值 \( \mu_k \)、协方差 \( \Sigma_k \) 描述。

    2. 🔁 EM 训练框架¶

    1. E 步：估计样本属于各成分的后验概率（责任度）。
    2. M 步：根据责任度更新模型参数。
    3. 重复迭代直到收敛或达到最大迭代次数。

    3. 💻 关键代码片段¶

    gmm = GaussianMixtureModel(
    n_components=args.n_components,
    max_iter=args.max_iter,
    tol=args.tol,
    random_state=args.random_state,
)
gmm.fit(X)

    gmm_upload/

   └─ iteration_log.csv

    🚀 快速开始¶

    1) 安装依赖¶

    pip install numpy matplotlib

    2) 运行实验¶

    python GMM.py --n-samples 1000 --n-components 3 --max-iter 100 --tol 1e-6 --random-state 42 --out-dir outputs

    如仅需保存文件、不弹窗：

    python GMM.py --no-show --out-dir outputs

    ⚙️ 参数配置说明¶

    参数 说明 默认值
    --n-samples 样本数量 1000
    --n-components 聚类成分数 3
    --max-iter EM 最大迭代轮数 100
    --tol 收敛阈值 1e-6
    --random-state 随机种子 42
    --out-dir 输出目录 outputs
    --no-show 不弹出图像窗口 关闭

    📈 输出结果¶

    1. 聚类对比图：cluster_comparison.png
    2. 收敛曲线：convergence_curve.png
    3. 迭代日志：iteration_log.csv

    日志文件记录了每轮的对数似然值，可用于判断 EM 是否趋于稳定。

    🛠️ 代码改动清单¶

    1) 数据生成支持随机种子参数¶

    改动：generate_data(n_samples, random_state)
    目的：保证实验可复现。

    2) 新增命令行参数入口¶

    改动：在 main 中加入 argparse。
    目的：避免反复修改源码参数。

    3) 图像支持自动保存¶

    改动：保存聚类图与收敛曲线到 outputs/。
    目的：便于实验留档和提交。

    4) 增加迭代日志导出¶

    改动：将每轮对数似然写入 iteration_log.csv。
    目的：可追踪收敛过程。
